**一、系统概述**

**1.1 系统背景**

智能搬运系统面向工厂/仓库产线 AGV 搬运场景，通过现场**物理呼叫器（多按钮终端）**触发物料搬运任

务，实现“工人一键呼叫、AGV 自动搬运”，减少人工沟通与派单成本。

**1.2 模块定位（核心一句话）**

**本系统核心功能：给呼叫器分配区域、配置每个按钮对应的搬运任务功能。**

**1.3 目标用户**

产线/仓库运维人员：新增、修改、维护呼叫器及按钮功能

生产管理员：查看配置、任务统计、在线状态

实施人员：现场部署、调试、批量配置

**二、整体功能结构**

系统围绕“**呼叫器-区域-按钮-任务**”四层管理：

区域管理：维护产线物理区域（如上料区、下料区、暂存区）

呼叫器管理：新增/编辑/删除呼叫器，绑定所属区域

按钮功能配置：给每个呼叫器的每个按钮分配功能（任务类型+目标位置）

任务与状态监控：查看呼叫记录、任务执行、在线状态**三、详细功能说明**

**3.1 区域管理**

**3.1.1 功能描述**

用于维护工厂所有物理区域，作为呼叫器和任务的“地理基础”。

**3.1.2 关键字段**

区域编号/名称：如“N 线下料区”“A 区上料区”

区域类型：上料区 / 下料区 / 暂存区 / 空框区

备注

**3.1.3 主要操作**

新增区域

编辑区域

删除区域（无呼叫器关联时）

列表查询

**3.2 呼叫器管理（终端管理）![3990df56d5d5cedd8c22adf5c5aca4f3](C:\Users\yixin\Documents\xwechat_files\wxid_7cup62w4b43822_2a08\temp\RWTemp\2026-05\9e20f478899dc29eb19741386f9343c8\3990df56d5d5cedd8c22adf5c5aca4f3.png)**

**3.2.1 功能描述**

管理现场所有物理呼叫器设备，为后续按钮配置做绑定。

**3.2.2 关键字段**

呼叫器编号（唯一）

呼叫器名称

所属区域（下拉选择“区域管理”中的数据）IP 地址、端口（通信参数）

在线状态（在线/离线）

启用状态（启用/停用）

**3.2.3 主要操作**

新增呼叫器：填写基础信息+绑定区域

编辑呼叫器：修改名称、区域、IP 等

删除呼叫器

批量导入/导出（便于批量部署）

按区域、状态筛选

**3.3 按钮功能配置（核心功能）**

**3.3.1 功能描述**

**给呼叫器的每一个物理按钮，分配具体的搬运任务功能。** 一个呼叫器通常有多个按钮（如按钮 1、按

钮 2、按钮 3），每个按钮可独立配置。

**3.3.2 可配置项（每个按钮）**

按钮序号：1 / 2 / 3 ……

按钮名称：自定义（如“满料呼叫”“空框返回”）

任务类型：

绑定（送料/配送）

解绑（空车返回/回收）

目标位置/工位：绑定到具体区域下的工位

启用/禁用

**3.3.3 配置示例**

呼叫器 A（下料区）按钮 1：满料配送 → 从下料区 → 上料区

按钮 2：空框返回 → 从暂存区 → 下料区

**3.3.4 操作流程**

在呼叫器列表点击“配置按钮”

进入按钮配置页

对每个按钮设置：名称+任务类型+目标位置

保存生效 → 现场按钮即可触发对应任务![3881f5a9ecb9cd251c2d9ce078776708](C:\Users\yixin\Documents\xwechat_files\wxid_7cup62w4b43822_2a08\temp\RWTemp\2026-05\9e20f478899dc29eb19741386f9343c8\3881f5a9ecb9cd251c2d9ce078776708.png)

**3.4 任务记录与呼叫日志**

**3.4.1 功能描述**

记录每个呼叫器、每个按钮的触发记录与任务结果，用于追溯与统计。

**3.4.2 关键字段**

呼叫时间

呼叫器编号+名称

按钮序号+名称

任务类型

任务状态：待执行/执行中/完成/异常

操作结果

**四、核心业务流程（配置→使用）**

管理员在【区域管理】建好产线区域

在【呼叫器管理】新增呼叫器，绑定对应区域

在【按钮配置】给每个按钮分配任务功能现场工人按下呼叫器按钮 → 系统生成搬运任务

AGV 执行任务，状态回写，日志留存

**五、系统价值**

灵活配置：按钮功能可随时调整，适配不同产线/工序

一键部署：支持批量导入，现场快速上线

责任清晰：呼叫器、按钮、区域全链路可追溯

降低沟通：工人一键呼叫，减少电话/对讲机沟通
